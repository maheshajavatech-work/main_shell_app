How This Modular App Works

1. Shell App

This is the main host that contains shared layout, navbar, and global styles. It dynamically loads micro-frontends based on route configuration.

2. Micro-Frontends (MFEs)

Each feature (Tasks, Reports, Settings) is built and deployed independently as an Angular app. These MFEs are lazy-loaded into the shell only when the route is accessed.

3. Remote Module Federation

Angular Module Federation loads routes or components remotely using the remoteEntry.js exposed by each MFE.

When you navigate to /reports, the shell app dynamically fetches remoteEntry.js from the Reports MFE running on localhost:4202.

The file exposes ./Module which contains the Angular routes for Reports. These routes are rendered inside the shell seamlessly.
